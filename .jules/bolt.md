## 2026-04-28 - Unmemoized JSON Array Filtering and Mapping in Top Level Views
**Learning:** Found a top-level view (`ModulesView`) that heavily re-filtered and mapped over the imported JSON `courses.modules` on every render (including minor local state changes like dark mode toggles or input changes). The JSON object is static, so these array methods only needed to run when the user's explicit `completed` state changed. Unmemoized inline arrow function props (`onOpenLesson={(id) => setActiveLesson(id)}`) also exacerbated re-renders.
**Action:** When working with large, static JSON datasets in React components, ensure that expensive derivative computations (like filtering arrays based on secondary state) are memoized using `useMemo` so they don't fire on unrelated UI state changes. Always ensure prop functions are either statically bound or appropriately memoized to avoid creating new references on every render.
