## 2024-05-03 - [Refactoring Nested Operations in React Component]
**Learning:** React functional components may unnecessarily repeat heavy list derivations using array functions like `map`, `filter`, and `reduce` consecutively on each render pass. Re-writing chains of array operations as a single loop can improve raw calculation speed, and wrapping it in `React.useMemo` memoizes these calculations, only forcing re-renders if underlying dependencies (like the `completed` mapping prop) are actually altered. Furthermore, identifying completely static derivations and extracting them out of the functional component boundary avoids performing identical static logic continuously.
**Action:** Always scan React components for repeated chained array operations on render passes. Evaluate what portions can be hoisted completely outside the component, and use `useMemo` in combination with single-pass iteration inside the component for variables derived from changing state or props.
